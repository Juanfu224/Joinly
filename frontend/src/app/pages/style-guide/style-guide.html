Componentes y patrones de diseño de Joinly para desarrollo consistente y accesible.

Comunicación entre Componentes

Sistema reactivo de comunicación entre componentes hermanos mediante CommunicationService. Implementa BehaviorSubject + Signals para estado compartido unidireccional.

Demostración en tiempo real

El componente emisor envía alertas (AlertService) y actualiza estado compartido (CommunicationService). El componente receptor escucha y muestra los cambios de estado y eventos automáticamente.

Patrones implementados:
  • ✅ BehaviorSubject: Estado persistente (usuario, filtros)
  • ✅ Subject: Eventos one-time sin persistencia
  • ✅ Signals (Angular 21): Reactividad nativa sin suscripciones manuales
  • ✅ Computed Signals: Estado derivado (isAuthenticated, hasActiveFilters)
  • ✅ takeUntilDestroyed(): Limpieza automática de suscripciones
  • ✅ Separación de responsabilidades: AlertService (UI) vs CommunicationService (estado)
  • ✅ OnPush compatible: Optimizado para detección de cambios

Casos de uso:

  • Estado de usuario compartido (header, sidebar, profile)
  • Filtros sincronizados entre componentes de búsqueda
  • Eventos del sistema (login, logout, CRUD operations)
  • Alertas visuales temporales (toasts via AlertService)

Theme Switcher

Sistema de cambio de tema claro/oscuro con persistencia en localStorage y detección automática de preferencias del sistema.

Componente Theme Toggle

Control programático

Tema claro Tema oscuro
Características del sistema de temas:
  • ✅ Persistencia: Guarda preferencia en localStorage
  • ✅ Detección automática: Lee preferencia del sistema operativo
  • ✅ Reactividad: Usa Signals de Angular para cambios instantáneos
  • ✅ Accesibilidad: Botón con aria-label dinámico
  • ✅ CSS Variables: Cambio de tema mediante data-theme en :root
  • ✅ Transiciones suaves: Animación de cambio de colores
  • ✅ Sincronización del sistema: Escucha cambios de prefers-color-scheme

Uso en componentes:

// Inyectar servicio
private readonly themeService = inject(ThemeService);

// Alternar tema
this.themeService.toggleTheme();

// Leer tema actual (Signal)
const currentTheme = this.themeService.currentTheme();

Alertas

Mensajes de alerta estáticos para feedback inline. Uso: confirmaciones, errores, advertencias, información contextual. Diferencia con toasts: estas son estáticas en la página, mientras que los toasts son temporales en la esquina de pantalla.

Tipos de alerta

Éxito: El registro se completó correctamente. Bienvenido a Joinly. Error: El correo electrónico ya está registrado. Intenta con otro. Advertencia: Tu suscripción expira en 3 días. Renueva ahora para no perder el acceso. Información: Revisa tus datos personales antes de guardar los cambios.

Alertas desechables

Las alertas pueden ser cerradas manualmente con el botón X cuando son dismissible.

7. Optimización Multimedia

Demostraciones de imágenes responsive con múltiples formatos (AVIF, WebP, JPG) y animaciones CSS optimizadas.

7.1 Componente FeatureImage

El componente FeatureImageComponent implementa imágenes responsive con:

  • srcset y sizes para diferentes tamaños de pantalla
  • Múltiples formatos de imagen (AVIF, WebP, JPG) con fallbacks automáticos
  • Art direction para mobile vs desktop con <picture>
  • Lazy loading nativo
  • Aspect ratio personalizable

Nota: Para ver las imágenes demo, necesitas generar las imágenes usando el script del plan con sharp-cli y colocarlas en public/assets/images/demo/.

7.3 Principios de Optimización

  • GPU-accelerated: Solo se animan transform y opacity
  • Duraciones óptimas: 150ms - 500ms para mejor UX
  • Will-change: Usado apropiadamente para preparar animaciones
  • Responsive: Imágenes optimizadas por tamaño de pantalla
  • Formatos modernos: AVIF (mejor compresión) con fallbacks
  • Lazy loading: Carga diferida de imágenes fuera del viewport
