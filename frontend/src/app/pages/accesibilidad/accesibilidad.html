Declaración de Accesibilidad

En Joinly nos comprometemos a hacer nuestra aplicación accesible para todas las personas, independientemente de sus capacidades. Esta página describe nuestro compromiso con la accesibilidad.

Nivel de conformidad

WCAG 2.1 AA

Joinly cumple con las Web Content Accessibility Guidelines (WCAG) 2.1 a nivel AA, el estándar reconocido internacionalmente para la accesibilidad web.

Características de accesibilidad implementadas

Navegación por teclado

  • Todos los elementos interactivos son accesibles mediante teclado
  • Indicadores de foco visibles (outline de 2px)
  • Modales con focus trap y cierre con Escape
  • Skip link para saltar al contenido principal
  • Orden de foco lógico en todos los flujos

Lectores de pantalla

  • HTML semántico (landmarks, headings, listas)
  • ARIA labels para elementos sin texto visible
  • Estados anunciados (expanded, pressed, invalid)
  • Live regions para notificaciones y errores
  • Un solo h1 por página con jerarquía correcta

Visuales

  • Contraste de colores mínimo 4.5:1 para texto
  • Información no transmitida solo por color
  • Tema oscuro completo con alto contraste
  • Tamaño mínimo táctil de 44x44px
  • Soporte para prefers-reduced-motion

Formularios

  • Labels asociados a cada campo
  • Errores anunciados automáticamente (role="alert")
  • Campos obligatorios indicados con aria-required
  • Descripciones de ayuda vinculadas con aria-describedby
  • Autocompletado habilitado donde corresponde

Compatibilidad con tecnología de asistencia

Joinly ha sido probado con las siguientes tecnologías de asistencia:

  • NVDA (Windows) con Chrome y Firefox
  • VoiceOver (macOS) con Safari
  • VoiceOver (iOS) con Safari
  • Navegación por teclado en Chrome, Firefox, Safari y Edge
