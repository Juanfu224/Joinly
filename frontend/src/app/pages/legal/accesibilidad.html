Declaración de Accesibilidad

Última actualización: 15 de febrero de 2026

1. Nuestro compromiso

En Joinly nos comprometemos a garantizar la accesibilidad digital para todas las personas, independientemente de sus capacidades. Estamos trabajando continuamente para mejorar la accesibilidad de nuestra plataforma web y garantizar que cumpla con los estándares internacionales.

Esta declaración de accesibilidad se aplica al sitio web joinly.studio.

2. Estándares de conformidad

Nos esforzamos por cumplir con las Web Content Accessibility Guidelines (WCAG) 2.1, nivel AA, definidas por el W3C. Estas directrices explican cómo hacer el contenido web más accesible para personas con discapacidades.

Implementamos las siguientes medidas de accesibilidad:

  • Navegación por teclado: Todo el contenido es accesible utilizando únicamente el teclado (Tab, Enter, Escape, flechas de navegación).
  • Skip link: Enlace para saltar directamente al contenido principal, evitando la navegación repetitiva.
  • Estructura semántica: Uso correcto de encabezados (H1-H6), landmarks HTML5 y listas para facilitar la navegación.
  • ARIA: Atributos ARIA apropiados para componentes interactivos (modales, menús, acordeones).
  • Texto alternativo: Todas las imágenes informativas incluyen descripciones textuales.
  • Contraste de colores: Los colores del texto y fondo cumplen con los ratios de contraste mínimos (4.5:1 para texto normal).
  • Focus visible: Indicadores visuales claros para el elemento que tiene el foco.
  • Formularios accesibles: Etiquetas asociadas a campos, mensajes de error descriptivos y validación clara.
  • Reduced motion: Respetamos las preferencias del sistema para reducir animaciones.

3. Compatibilidad con tecnologías asistivas

Joinly está diseñado para ser compatible con las siguientes tecnologías de asistencia:

  • Lectores de pantalla (NVDA, JAWS, VoiceOver, Narrador)
  • Navegación solo con teclado
  • Herramientas de ampliación de pantalla
  • Tecnologías de reconocimiento de voz

Hemos probado la accesibilidad con los siguientes navegadores:

  • Google Chrome (última versión)
  • Mozilla Firefox (última versión)
  • Safari (última versión)
  • Microsoft Edge (última versión)

4. Limitaciones conocidas

A pesar de nuestros esfuerzos, algunas partes del sitio pueden no ser completamente accesibles:

  • Algunos componentes de terceros podrían no cumplir totalmente con WCAG 2.1 AA
  • Los documentos PDF externos podrían no estar etiquetados correctamente

Estamos trabajando para resolver estas limitaciones y mejorar continuamente la accesibilidad de nuestra plataforma.

6. Enfoque de mejora continua

La accesibilidad es un proceso continuo, no un destino final. Nuestro equipo de desarrollo:

  • Realiza auditorías de accesibilidad de forma regular

  • Implementa mejoras basadas en el feedback de los usuarios
